## 发现的问题

### 问题1：货币转换错误（已修复）

**问题描述**：
- 商品成本价是CNY（例如197.00 CNY）
- 订单货币是USD
- 订单同步时，直接将CNY的成本价当作USD存储（例如197.00 CNY被存储为197.00 USD）
- 导致成本价被错误放大7.1倍（197.00 USD = 1398.70 CNY，而不是197.00 CNY）

**影响**：
- SKU `LBB2-1-US` 有182个USD订单使用了错误的成本价（196.00 USD = 1391.60 CNY）
- 这些订单的利润为负（-1178.60 CNY）

**修复**：
- 已修复订单同步逻辑，根据订单货币和成本价货币进行正确转换
- 如果成本价是CNY，订单货币是USD，会将成本价转换为USD（197.00 CNY / 7.1 = 27.75 USD）

### 问题2：历史订单成本价错误

**问题描述**：
- 订单时间在2025-10-11到2025-11-21之间
- 成本记录生效时间是2025-11-21
- 订单同步时找不到对应的成本价，但订单中却存储了错误的成本价（196.00 USD）

**解决方案**：
- 需要为历史订单时间创建成本记录，或重新同步这些订单
- 或者手动更新这些订单的成本和利润
